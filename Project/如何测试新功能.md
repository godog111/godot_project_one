# 如何测试修复后的智能堆叠卡牌系统

## 问题回顾
之前进入牌桌测试时报错：`错误 (214, 9)： Function "update_stack_visual()" not found in base self.`

## 修复已完成 ✅
该错误已修复，现在可以正常测试。

## 测试步骤（最简单的方法）

### 方法一：一键测试（推荐）
1. **打开 Godot 编辑器**
2. **在文件系统中找到**：`launch_real_cards_test.tscn`
3. **双击运行**这个场景
4. **点击屏幕上的"开始测试"按钮**（或按**空格键**）
5. **查看控制台输出**，确认系统状态

### 方法二：直接快速测试
1. **直接运行**：`quick_test_real_cards.tscn`
2. **系统自动执行**完整测试流程
3. **查看控制台**了解测试结果

## 测试时观察什么

### 1. 控制台输出（最重要）
运行后，查看控制台是否显示：
```
✓ CardInfo单例已找到
✓ 卡牌数据已加载 (XX 条记录)
✓ 真实卡牌场景加载成功
✓ 卡牌生成成功
✓ update_stack_visual方法存在
```

### 2. 屏幕显示
- 屏幕上应显示**5张测试卡牌**
- 卡牌位置应**水平排列**
- 卡牌应有**不同的颜色**（表示不同类型）

### 3. 功能测试
1. **点击并拖动**一张卡牌
2. **拖动到另一张卡牌附近**（距离约100像素内）
3. **观察是否自动吸附**
4. **松开鼠标**，卡牌应停留在吸附位置

## 快捷键说明
- **F1**：显示调试信息（卡牌位置、类型）
- **空格**：重新生成卡牌
- **ESC**：退出测试

## 如果测试失败怎么办

### 情况1：看不到卡牌
1. 检查控制台是否有错误信息
2. 确保运行的是 `launch_real_cards_test.tscn` 场景
3. 按**空格键**重新生成卡牌

### 情况2：拖动没有吸附效果
1. 确保拖动卡牌到足够近的距离
2. 不同类型的卡牌吸附偏移不同
3. 按**F1**查看卡牌类型和位置

### 情况3：出现其他错误
1. 查看错误信息的具体行号
2. 检查是否运行了正确的测试场景
3. 按**ESC**退出，重新开始
