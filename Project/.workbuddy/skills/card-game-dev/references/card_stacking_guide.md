# 卡牌堆叠系统开发指南

## 概述
卡牌堆叠系统允许玩家将多张卡牌堆叠在一起，形成有序的卡牌组。这在许多卡牌游戏中是常见功能，如扑克牌、集换式卡牌游戏等。

## 核心概念

### 1. 堆叠检测
- **距离检测**：当两张卡牌的距离小于阈值时自动堆叠
- **区域检测**：在特定区域内的卡牌自动形成堆叠
- **手动堆叠**：玩家拖动卡牌到另一张卡牌上

### 2. 堆叠管理
- **堆叠ID**：每个堆叠有唯一标识符
- **堆叠顺序**：保持卡牌在堆叠中的顺序
- **堆叠布局**：自动计算卡牌在堆叠中的位置

### 3. 堆叠交互
- **拾取堆叠**：点击堆叠顶部卡牌拾取整个堆叠
- **拆分堆叠**：从堆叠中拖出单张卡牌
- **合并堆叠**：将一个堆叠放到另一个堆叠上

### 步骤2：实现堆叠检测
```gdscript
func can_stack(card_a, card_b) -> bool:
    var distance = card_a.global_position.distance_to(card_b.global_position)
    return distance < STACK_THRESHOLD
```

### 步骤3：实现堆叠布局
```gdscript
func update_stack_layout(stack_id):
    var cards = active_stacks[stack_id]
    for i in range(cards.size()):
        var offset_y = i * STACK_SPACING
        cards[i].target_position = base_position - Vector3(0, offset_y, 0)
```

## 性能优化建议

### 1. 空间分区
- 使用网格或四叉树进行空间分区
- 只检查相邻单元格中的卡牌
- 减少距离计算次数

### 2. 对象池
- 重用堆叠容器节点
- 预分配内存用于常见堆叠大小
- 避免频繁的内存分配

### 3. 渲染优化
- 对堆叠中的卡牌使用实例化渲染
- 只渲染可见的卡牌部分
- 使用LOD（细节层次）系统

## 常见问题解决方案

### 问题1：堆叠卡牌闪烁
**原因**：位置更新冲突
**解决方案**：使用tween动画平滑过渡，避免直接设置位置

### 问题2：堆叠检测不准确
**原因**：距离阈值设置不当
**解决方案**：使用卡牌边界框进行精确碰撞检测

### 问题3：性能问题
**原因**：每帧检查所有卡牌对
**解决方案**：实现惰性检测，只在卡牌移动时检查
