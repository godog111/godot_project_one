# 卡牌游戏开发Skill

## 目的
这个Skill提供卡牌游戏开发的专业知识和代码模板，帮助WorkBuddy快速构建高质量的Godot卡牌游戏项目。

## 何时使用
- 当用户需要开发卡牌游戏相关功能时
- 当需要实现卡牌堆叠、旋转、缩放等交互功能时
- 当需要边界检测、区域管理等游戏逻辑时
- 当需要优化卡牌渲染和性能时

## 核心功能
1. **卡牌堆叠系统**：自动堆叠检测、堆叠布局管理
2. **卡牌变换系统**：旋转、缩放、动画过渡
3. **边界管理系统**：游戏区域划分、边界检测
4. **输入处理系统**：多设备支持（鼠标、键盘、触摸屏）
5. **状态管理系统**：卡牌状态保存和加载

## 使用方法
### 1. 卡牌堆叠
当需要实现卡牌堆叠功能时：
- 参考`references/card_stacking_guide.md`了解堆叠原理
- 使用`scripts/generate_stacking_system.py`生成堆叠管理器代码
- 查看`assets/card_stacking_example.tscn`示例场景

### 2. 卡牌变换
当需要实现卡牌旋转和缩放时：
- 使用`scripts/generate_transform_system.py`生成变换系统
- 参考`references/transform_guide.md`了解多设备输入处理

### 3. 边界管理
当需要实现游戏区域限制时：
- 使用`scripts/generate_boundary_system.py`生成边界系统
- 查看`assets/boundary_example.gd`示例脚本

## 工作流程
1. 分析现有项目结构和需求
2. 选择合适的代码模板或生成器
3. 集成生成的代码到现有项目中
4. 测试功能并优化性能
5. 生成文档和测试报告

## 最佳实践
1. **模块化设计**：保持各个系统独立，便于维护
2. **事件驱动**：使用信号系统实现松耦合
3. **性能优化**：避免不必要的渲染，使用对象池
4. **可扩展性**：设计时考虑未来功能扩展
5. **用户体验**：确保交互流畅，响应及时
