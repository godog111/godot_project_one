# 2026-03-23 工作日志

## 牌桌网格吸附系统实现

### 需求
用户要求对 card_table_main 场景实现网格锚点吸附系统，目标效果参考图：
- 固定间距二维锚点网格（间距 = 卡牌宽/高的 1/2，可配置）
- 卡牌松手时左上角吸附到最近的空锚点
- 同名卡牌合并（doStack），异名占用则 BFS 找下一个空锚点
- 桌面可以中键拖动，锚点随桌面偏移

### 新增/修改文件
- `cards/grid_snap_table.gd`（新建）：GridSnapTable 基类，含锚点字典、place_card、release_card、add_card 兼容、桌面拖动、update_weight 空兼容
- `cards/card_table_main.gd`（重写）：继承 GridSnapTable，负责初始卡牌生成、CardInfo 加载、键盘快捷键
- `cards/card_table_main.tscn`（更新）：引用新脚本，添加 snap_step_x/y 导出参数
- `cards/draggable_card_with_smart_stacking.gd`（修改）：松手时调用 _find_deck_under_mouse + add_card；按下时通知 preDeck.release_card 释放锚点

### 关键设计
- GridSnapTable 用 `_anchor_map: Dictionary[Vector2i -> card]` 管理锚点
- BFS 搜索（Chebyshev 环形扩展）找最近可用锚点
- add_card 包装 place_card，兼容旧 card.gd 接口
- update_weight 空实现兼容旧代码调用

## 修复：拖动堆叠卡牌数量重置问题

**问题原因**：`card.gd` 的 `_on_button_button_down` 在拖动时把 `num` 设为 1（旧 deck 的拆分堆叠逻辑），导致堆叠数量丢失。

**修复方案**：检测 `preDeck` 是否有 `release_card` 方法（GridSnapTable 特征），有则走新模式：
- 调用 `release_card` 释放锚点
- 不拆分 `num`，整体拖动
- 保留旧逻辑兼容旧 deck

## 修复：AllButton 点击触发拖动

**问题原因**：`_on_all_button_button_down` 方法会在点击数字标签时触发拖动（旧 deck 的拆分功能）。

**修复方案**：在 GridSnapTable 模式下直接返回，禁用此功能。

## 修复：偶发 Invalid call 'add_card' 错误

**问题原因**：`whichDeckMouseIn` 或 `preDeck` 可能被设置为非 GridSnapTable 节点（如旧的 deck 或普通 Control）。

**修复方案**：调用 `add_card` 前先用 `has_method("add_card")` 检查。

## 修改：锚点只在牌桌内生成 + 无位置时回退

**问题**：牌桌大小调整后，锚点生成超出牌桌范围；卡牌找不到位置时随便放置。

**修复方案**：
1. `_find_available_anchor`：计算牌桌网格范围 `min_grid` / `max_grid`，只搜索范围内的锚点
2. `release_card`：记录卡牌拖起时的位置到 meta `_last_valid_position`
3. `place_card`：找不到可用锚点时，调用 `_place_at_position` 回到原位
4. 新增 `_place_at_position` 方法：不绑定锚点，只是动画移动到指定位置
5. `_spawn_initial_cards`：计算牌桌可容纳的网格范围，超出范围的卡牌跳过生成

**关键修复**：网格范围计算从 `Vector2.ZERO` 改为 `global_position`，确保只限制在 `CardTableMain` 节点范围内，而不是整个屏幕
